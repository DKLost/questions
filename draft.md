# 关于本软件中知识点的实现

任意一条知识点都被视作一条构造。

一条构造由四部分组成：中心数据，方式存在，注入存在，构造结果。

一条构造可以理解为对中心P以方式F注入存在X，从而构造一个仍是P的结果P'的过程。

对一个知识点的复习，即被视作再引发一条构造，并判断引发结果是否正确的过程。

通常题干-答案式的知识记录，答案本身便是一种对构造结果的引起，而构造过程，即方式、中心、注入三者则被隐含在了题干当中。

为了方便描述，可以将任意一条构造都可以转化描述为：P'是将中心P的Y确定为X的P。其实质是找到一个方式为.Y=的等价构造。

例如：

问题：在水溶液中或熔融状态下能够电离出离子的化合物是什么？。

答案：电解质。

其本身可被视作这样的一条构造：

1. 中心数据：在水溶液中或熔融状态下能够电离出离子的某物
2. 方式存在：确定某物为
3. 注入存在：化合物
4. 结果存在：电解质

为了方便描述，可以将其写作：在水溶液中或熔融状态下能够电离出离子的某物[确定某物为: 化合物] = 电解质

- 其也可被视作这样一条构造：化合物[确定特征为: 在水溶液中或熔融状态下能够电离出离子] = 电解质

- 该构造也可被改写为：化合物[确定特征为: A的构造结果] = 电解质

  - 构造A：能够电离出离子[确定发生条件为: 在水溶液中或熔融状态下] = 在水溶液中或熔融状态下能够电离出离子

  - 构造A又可以被改写为：能够电离出离子[确定发生条件为: B构造结果] = 在水溶液中或熔融状态下能够电离出离子
    - 构造B：在某情况下[确定某情况为: C的构造结果] = 在水溶液中或熔融状态下
    - 构造C：D的构造结果[确定第二参数为: 熔融状态]  = 水溶液中或熔融状态
    - 构造D：...或...[确定第一参数为: 水溶液中] = 水溶液中或...

我们可以把任意的知识点不断递进细分成尽可能小的构造的组合，从而便于机器理解不同存在之间的可能关系。

为了减轻构造的难度，目前采取分离式设计，即答案本身之时只挂钩到某一构造上，但构造的中心、方式、数据、结果都为空。

# 题干-答案

任何一条题干-答案，都可以视为有3个构造组成：

题干本身引发出一个存在，此时有构造1

从构造1的构造结果，确定其对应的文字形式的存在的过程，此时有构造2

从答案承载的文字数据，确定其对应的文字形式的存在，此时有构造3

构造1：

1. 中心：题干隐含
2. 方式：题干隐含
3. 注入：题干隐含
4. 结果：某存在A

构造2：

1. 中心：空文字存在
2. 方式：确定所指为
3. 注入：A
4. 结果：文字存在B

构造3：

1. 中心：答案承载的文字数据
2. 方式：是
3. 注入：存在
4. 结果：文字存在B（答案承载的文字数据的存在）

# 笔记格式

完整格式：

A

- ~F~X ⇒ R: M //相当于(<<2 (<<1 R (~F X A)) M)	
  - ~G~Y ⇒ T: N //相当于(<<2 (<<1 T (~G Y X')) N)，其中(<<2 (<<1 == X') (~F X A))
    - ...
  - ...
- ...

简写格式:

当省略时，F默认为“的”；X默认为“self”；R默认为“是”；M通常和R一起省略，相当于“是: X”。

A

- ~F~X ⇒ R: M //相当于(<<2 (<<1 R (~F X A)) M)
- X ⇒ R: M //省略~F~，等价于~的~X ⇒ R: M，相当于(<<2 (<<1 R (确定约束者为 X A)) M)
- ~F~ ⇒ R: M //省略X，等价于~F~self⇒ R: M，相当于(<<2 (<<1 R (~F self A)) M)
- ~F~X ⇒ M //省略R: ，等价于~F~X ⇒ 是: M，相当于(<<2 (<<1 := (~F X A)) M)
- X ⇒ M //省略~F~ R: ，等价于~的~X ⇒ 是: M，相当于(<<2 (<<1 := (~确定约束者为 X A)) M)
- ~F~X //省略⇒ R: M，等价于~F~X ⇒ 是: X，相当于(<<2 (<<1 := (~F X A)) X)，该格式必须有子项
- X  //省略~F~ ⇒ R: M，等价于~的~X ⇒ 是: X，相当于(<<2 (<<1 := (确定约束者为 X A)) X)，该格式必须有子项
- R: M //省略~F~X ，等价于~的~self ⇒ R: M，相当于(<<2 (<<1 R A) M)，该格式的子项是对M的描述
- R: //省略~F~X M，等价于~的~self ⇒ R: {}，相当于(<<2 (<<1 R A) {})，该格式必须有子项，该格式的子项是对M的描述
- ⇒ M //省略~F~X ⇒ R:，等价于~的~self ⇒ 是: M，相当于(<<2 (<<1 := A) M)，该格式的子项是对M的描述

1. R: M //等价于~的~第n存在 ⇒ R: M，相当于(<<2 (<<1 R (确定约束者为 (确定序号为 {} 1) A)) M)
2. M //等价于~的~第n存在 ⇒ 是: M，相当于(<<2 (<<1 := (确定约束者为 (确定序号为 {} 1) A)) M)

若M是(~G N A)，则可表示成~G~*N*，其中只用*N*是等价于~的~*N*，即相当于(确定约束者为 N A)

若R是A，则可表示成*A*

"~的~X"，也可写作.X或~→~X，都表示(确定约束者为 X A)

## 举例

对于“3大于2”这一知识点，可以描述为：

3

- 大于：2 //即 ~的~self ⇒ 大于：2

2

- 被大于：3 //即 ~的~self ⇒ 被大于：3

大于

- ~的左操作数是~3 ⇒ *大于*：~的右操作数是~*2*



## 转写规则

对于任意构造(<<2 (<<1 R (F P X)) Q)，都可以转写为6种描述为：

### 1.X为主题

X

- ~~F~P ⇒ R: Q //(<<2 (<<1 R (F P X)) Q)

### 2.Q为主题

Q

- ~→~self ⇒ ~R: (F P X)

### 3.R为主题

R

- ~的[A]是~(F P X) ⇒ *R* : ~的[B]是~*Q* // [A] [B]表示R的第一参数和第二参数，即可以有(R '[A] [B]) := [A]

### 4.R为主题

F

- ~的[A]是~(F P X) ⇒ *F* : ~的[B]是~*X*

### 5.P'为主题(即(F P X)为主题)

 (F P X)

- ~→~self ⇒ R: X

### 6.P为主题

P

- ~G~self ⇒ R: Q //其中 (~G self P)  = (F P X) ， (<<2 (<<1 R (~G self P) )) Q)
- ~FX的~self ⇒ R: Q

举例：

(<<2 (<<1 := (自 颜色 苹果)) 红色)，~自 = 的，~的 = 自

苹果

- ~的~颜色 ⇒ 是：红色
- ~→~颜色 ⇒ 是：红色 //→表示的

颜色

- ~自苹果的~self ⇒ 是：红色
- ~←苹果→~self ⇒ 是: 红色 //←表示自





## 表格化

- A
  - ~F~X ⇒ R: M
  - ~g~Y ⇒ ...
- B
  - ~F~X ⇒ T: N
  - ~g~Y ⇒ ...

可表格化为（其中⇒可省略）

|        | A    | B    |
| ------ | ---- | ---- |
| ~F~X ⇒ | R: M | T: N |
| ~g~Y ⇒ | ...  | ...  |

- A
  - ~F~X ⇒ R: M
- B
  - ~F~X ⇒ R: N

可表格化为（其中：不可省略）

|           | A    | B    |
| --------- | ---- | ---- |
| ~F~X ⇒ R: | M    | N    |



## 通用构造

对于任意一句话S，如果想将对其中某个存在E作为主题，都可以如此构造：

E

- 作为[A]参与到: 'S //'S是S中E所在的部分用[A]表示后的结果
- 'S //直接用[...]在'S中表示[A]

举例：以3为主题，描述3大于2这一知识点

3

- 大于：2 //即 ~的~self ⇒ 大于：2，(<<2 (<<1 大于 3) 2)
- 作为[A]参与到: [A]大于2，(<<2 (<<1 作为[A]参与到 3) [A]大于2)
- [...]大于2 //即作为[A]参与到: [A]大于2，(<<2 (<<1 作为[A]参与到 3) [A]大于2)

粒子之间的作用力

- 作为[A]参与到: 晶体分类的依据是[A]
- 晶体分类的依据是[...]
